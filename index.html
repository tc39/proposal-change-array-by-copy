    1.1.1.1 Array.prototype.popped ( )

    When the popped method is called, the following steps are taken:

    1. Let O be ? ToObject(this value).
    2. Let len be ? LengthOfArrayLike(O).
    3. Let newLen be max(len - 1, 0).
    4. Let A be ? ArraySpeciesCreate(O, 𝔽(newLen)).
    5. Let k be 0.
    6. Repeat, while k < newLen,
      1. Let Pk be ! ToString(𝔽(k)).
      2. Let elementK be ? Get(O, Pk).
      3. Perform ? CreateDataPropertyOrThrow(A, Pk, elementK).
      4. Set k to k + 1.
    7. Return A.

    1.1.1.2 Array.prototype.pushed ( ...items )

    When the pushed method is called, the following steps are taken:

    1. Let O be ? ToObject(this value).
    2. Let itemCount be the number of elements in items.
    3. Let len be ? LengthOfArrayLike(O).
    4. let newLen be len + itemCount.
    5. Let A be ? ArraySpeciesCreate(O, 𝔽(newLen)).
    6. Let k be 0.
    7. Repeat, while k < len,
      1. Let Pk be ! ToString(𝔽(k)).
      2. Let elementK be ? Get(O, Pk).
      3. Perform ? CreateDataPropertyOrThrow(A, Pk, elementK).
      4. Set k to k + 1.
    8. For each element E of items, do
      1. Assert: k < newLen.
      2. Let Pk be ! ToString(𝔽(k)).
      3. Perform ? CreateDataPropertyOrThrow(A, Pk, E).
      4. Set k to k + 1.
    9. Return A.

    1.1.1.5 Array.prototype.sorted ( compareFn )

    When the sorted method is called, the following steps are taken:

    1. If comparefn is not undefined and IsCallable(comparefn) is false, throw a TypeError exception.
    2. Let O be ? ToObject(this value).
    3. Let len be ? LengthOfArrayLike(O).
    4. Let items be a new empty List.
    5. Let k be 0.
    6. Repeat, while k < len,
      1. Let Pk be ! ToString(𝔽(k)).
      2. Let kValue be ? Get(O, Pk).
      3. Append kValue to items.
      4. Set k to k + 1.
    7. Sort items using an implementation-defined sequence of calls to SortCompare. If any such call returns an abrupt completion, stop before performing any further calls to SortCompare or steps in this algorithm and return that completion.
    8. Let A be ? ArraySpeciesCreate(O, 𝔽(len)).
    9. Let j be 0.
    10. For each element E of items, do
      1. Let Pj be ! ToString(𝔽(j)).
      2. Perform ? CreateDataPropertyOrThrow(A, Pj, E).
      3. Set j to j + 1.
    11. Return A.

    1.1.1.6 Array.prototype.spliced ( start, deleteCount, ...items )

    When the spliced method is called, the following steps are taken:

    1. Let O be the this value.
    2. Let len be ? LengthOfArrayLike(O).
    3. Let relativeStart be ? ToIntegerOrInfinity(start).
    4. If relativeStart is -∞, let actualStart be 0.
    5. Else if relativeStart < 0, let actualStart be max(len + relativeStart, 0).
    6. Else, let actualStart be min(relativeStart, len).
    7. If start is not present, then
      1. Let insertCount be 0.
      2. Let actualDeleteCount be 0.
    8. Else if deleteCount is not present, then
      1. Let insertCount be 0.
      2. Let actualDeleteCount be len - actualStart.
    9. Else,
      1. Let insertCount be the number of elements in items.
      2. Let dc be ? ToIntegerOrInfinity(deleteCount).
      3. Let actualDeleteCount be the result of clamping dc between 0 and len - actualStart.
    10. Let newLen be len + insertCount_ - actualDeleteCount.
    11. Let A be ? ArraySpeciesCreate(O, 𝔽(newLen)).
    12. Let k be 0.
    13. Repeat, while k < newLen,
      1. Let Pk be ! ToString(𝔽(k)).
      2. Let kValue be ? Get(O, Pk).
      3. Perform ? CreateDataPropertyOrThrow(A, Pk, kValue).
      4. Set k to k + 1.
    14. Let k be actualStart.
    15. For each element E of items, do
      1. Let Pk be ! ToString(𝔽(k)).
      2. Perform ? CreateDataPropertyOrThrow(A, Pk, E).
      3. Set k to k + 1.
    16. Repeat, while k < newLen,
      1. Let Pk be ! ToString(𝔽(k)).
      2. Let from be ! ToString(𝔽(k + actualDeleteCount - insertCount)).
      3. Let fromValue be ? Get(O, from).
      4. Perform ? CreateDataPropertyOrThrow(A, Pk, fromValue).
      5. Set k to k + 1.
    17. Return A.

    1.1.1.8 Array.prototype.with ( index, value )

    When the with method is called, the following steps are taken:

    1. Let O be the this value.
    2. Let len be ? LengthOfArrayLike(O).
    3. If IsIntegralNumber(index) is false, throw a RangeError exception.
    4. If index ≥ len, throw a RangeError exception.
    5. If index < 0, let actualIndex be len + index.
    6. Else, let actualIndex be index.
    7. If actualIndex < 0, throw a RangeError exception.
    8. Let A be ? ArraySpeciesCreate(O, 𝔽(len)).
    9. Let k be 0.
    10. Repeat, while k < len,
      1. Let Pk be ! ToString(𝔽(k)).
      2. If k is actualIndex, let fromValue be value.
      3. Else, let fromValue be ? Get(O, Pk).
      4. Perform ? CreateDataPropertyOrThrow(A, Pk, fromValue).
      5. Set k to k + 1.
    11. Return A.

    1.2 TypedArray Objects

    1.2.1 Abstract Operations for TypedArray Objects

    1.2.1.1 TypedArrayCopyRange ( src, target, start, limit )

    The abstract operation TypedArrayCopyRange takes arguments src (a TypedArray object), target (a TypedArray object), start (a non-negative integer), and limit (a non-negative integer). It performs the following steps when called:

    1. Assert: src is an Object that has [[TypedArrayName]] and [[ContentType]] internal slots.
    2. Assert: target is an Object that has [[TypedArrayName]] and [[ContentType]] internal slots.
    3. Assert: start is a non-negative integer.
    4. Assert: limit is a non-negative integer.
    5. Assert: limit is ≤ the length of src.
    6. Assert: limit is ≤ the length of target.
    7. Let srcConstructor be ? SpeciesConstructor(src, defaultConstructor).
    8. Let targetConstructor be ? SpeciesConstructor(target, defaultConstructor).
    9. Assert: srcConstructor is targetConstructor.
    10. Let k be start.
    11. Repeat, while k < limit,
      1. Let Pk be ! ToString(𝔽(k)).
      2. Let kValue be ! Get(src, Pk).
      3. Perform ? Set(target, Pk, kValue, true).
      4. Set k to k + 1.

    1.2.2.1.1 %TypedArray%.prototype.popped ( )

    When the popped method is called, the following steps are taken:

    1. Let O be the this value.
    2. Perform ? ValidateTypedArray(O).
    3. Let len be O.[[ArrayLength]].
    4. Let newLen be max(len - 1, 0).
    5. Let A be ? TypedArraySpeciesCreate(O, « 𝔽(newLen) »).
    6. Perform ? TypedArrayCopyRange(O, A, 0𝔽, 𝔽(newLen)).
    7. Return A.

      3. Perform ? Set(A, Pk, E, true).

    2. Let O be the this value.
    3. Perform ? ValidateTypedArray(O).
    4. Let len be O.[[ArrayLength]].
    5. Let items be a new empty List.

    9. Let A be ? TypedArraySpeciesCreate(O, « 𝔽(len) »).

    The following version of SortCompare is used by %TypedArray%.prototype.sorted.

    The abstract operation TypedArraySortCompare performs the following steps when called:

    1. Assert: Both Type(x) and Type(y) are Number or both are BigInt.
    2. If comparefn is not undefined, then
      1. Let v be ? ToNumber(? Call(comparefn, undefined, « x, y »)).
      2. If IsDetachedBuffer(buffer) is true, throw a TypeError exception.
      3. If v is NaN, return +0𝔽.
      4. Return v.
    3. If x and y are both NaN, return +0𝔽.
    4. If x is NaN, return 1𝔽.
    5. If y is NaN, return -1𝔽.
    6. If x < y, return -1𝔽.
    7. If x > y, return 1𝔽.
    8. If x is -0𝔽 and y is +0𝔽, return -1𝔽.
    9. If x is +0𝔽 and y is -0𝔽, return 1𝔽.
    10. Return +0𝔽.

    1.2.2.1.6 %TypedArray%.prototype.spliced ( start, deleteCount, ...items )

    When the spliced method is called, the following steps are taken:

    1. Let O be the this value.
    2. Perform ? ValidateTypedArray(O).
    3. Let len be O.[[ArrayLength]].
    4. Let relativeStart be ? ToIntegerOrInfinity(start).
    5. If relativeStart is -∞, let actualStart be 0.
    6. Else if relativeStart < 0, let actualStart be max(len + relativeStart, 0).
    7. Else, let actualStart be min(relativeStart, len).
    8. If start is not present, then
      1. Let insertCount be 0.
      2. Let actualDeleteCount be 0.
    9. Else if deleteCount is not present, then
      1. Let insertCount be 0.
      2. Let actualDeleteCount be len - actualStart.
    10. Else,
      1. Let insertCount be the number of elements in items.
      2. Let dc be ? ToIntegerOrInfinity(deleteCount).
      3. Let actualDeleteCount be the result of clamping dc between 0 and len - actualStart.
    11. Let newLen be len + insertCount_ - actualDeleteCount.
    12. Let A be ? TypedArraySpeciesCreate(O, « 𝔽(newLen) »).
    13. Perform ? TypedArrayCopyRange(O, A, 0𝔽, 𝔽(actualStart)).
    14. Let k be actualStart.
    15. For each element E of items, do
      1. Let Pk be ! ToString(𝔽(k)).
      2. Perform ? Set(A, Pk, E, true).
      3. Set k to k + 1.
    16. Repeat, while k < newLen,
      1. Let Pk be ! ToString(𝔽(k)).
      2. Let from be ! ToString(𝔽(k + actualDeleteCount - insertCount)).
      3. Let fromValue be ! Get(O, from).
      4. Perform ? Set(A, Pk, fromValue, true).
      5. Set k to k + 1.
    17. Return A.

    1.2.2.1.8 %TypedArray%.prototype.with ( index, value )

    When the with method is called, the following steps are taken:

    1. Let O be the this value.
    2. Perform ? ValidateTypedArray(O).
    3. Let len be O.[[ArrayLength]].
    4. If IsIntegralNumber(index) is false, throw a RangeError exception.
    5. If index < 0, let actualIndex be len + index.
    6. Else, let actualIndex be index.
    7. If ! IsValidIntegerIndex(O, actualIndex) is false, throw a RangeError exception.
    8. Let A be ? TypedArraySpeciesCreate(O, « 𝔽(len) »).
    9. Let k be 0.
    10. Repeat, while k < len,
      1. Let Pk be ! ToString(𝔽(k)).
      2. If k is actualIndex, let fromValue be value.
      3. Else, let fromValue be ! Get(O, Pk).
      4. Perform ? Set(A, Pk, fromValue, true).
      5. Set k to k + 1.
    11. Return A.
